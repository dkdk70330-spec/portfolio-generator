# AI 캐릭터 포트폴리오 생성기 프로토타입

## 포함 기능

### 제작자
- 사이트 제목과 설명
- 프로필 PNG
- 제작자 이름, 아이디, 소개 문단
- 관리자 제공 소셜 서비스 선택 및 URL 입력

### 세계관
- 추가, 수정, 삭제
- 순서 변경
- 대표 PNG
- 이름, 부제, 사용자 태그, 소개 문단
- 추가 섹션 여러 개
- 연결된 캐릭터 표시

### 캐릭터
- 추가, 수정, 삭제, 복제
- 순서 변경
- 대표 캐릭터 최대 3명
- 세계관 선택 또는 독립 캐릭터
- 이름, 부제, 소개 문단
- 관리자 제공 장르 선택
- 사용자 태그 자유 입력
- PNG 이미지 최대 5장 및 순서 변경
- 관리자 제공 플랫폼 선택 및 플랫폼별 URL
- 추가 콘텐츠 여러 개
- 스포일러 여부와 경고문

### 미리보기
- 전체 캐릭터 보기
- 블랙 테마
- 화이트 테마

## 관리자와 사용자 데이터 분리

관리자만 수정:
- `shared/admin-catalog.js`
- 소셜 서비스 ID, 이름, 로고
- 플랫폼 ID, 이름, 로고
- 장르 목록

사용자가 생성기에서 수정:
- `window.EMPTY_PROJECT` 구조에 해당하는 프로젝트 데이터
- 제작자 정보
- 선택한 소셜 링크와 URL
- 세계관
- 캐릭터
- 사용자 태그
- 콘텐츠
- 업로드 PNG

관리자 카탈로그 객체는 `deepFreeze`로 잠겨 있으며 생성기 화면에는
관리자 서비스, 플랫폼, 장르를 추가하거나 삭제하는 기능이 없습니다.

## GitHub Pages 구조

이 ZIP의 파일을 저장소 루트에 배치합니다.

```text
저장소/
├── index.html
├── generator.css
├── generator.js
└── shared/
    ├── admin-catalog.js
    └── empty-project.js
```

관리자 로고를 표시하려면 기존 템플릿 이미지가 아래 위치에 있어야 합니다.

```text
template/images/profile-links/
template/images/platforms/
```

이미지가 없더라도 서비스명의 첫 글자가 대체 아이콘으로 표시됩니다.

## 프로토타입 제한

- 입력 데이터는 현재 브라우저 탭의 세션에서만 유지됩니다.
- 업로드 PNG는 캔버스로 다시 인코딩되어 원본 파일명과 PNG 메타데이터를 보관하지 않습니다.
- 아직 프로젝트 저장, 불러오기, 사이트 ZIP 내보내기는 포함하지 않습니다.
- 미리보기는 생성기 내부 프로토타입이며 최종 템플릿 iframe 연결 전 단계입니다.
